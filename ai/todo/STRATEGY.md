# Kokukoku — 戦略ドキュメント

## 1. 名前の由来

### Kokukoku（刻々）

名前には三つの層がある。

**第1層：表の意味**
「刻々」— 刻一刻と、一瞬一瞬が過ぎていくさま。時は止まらず流れ続ける。

**第2層：江戸の時間**
「刻（こく）」は江戸時代の時間単位。一刻＝約2時間。人々は不定時法で、日の出と日の入りに合わせて自然なリズムで暮らしていた。Pomodoroが25分を一区切りにするように、江戸の人々は「一刻」を一区切りにしていた。

**第3層：刻む**
「刻」のもう一つの意味は「彫り刻む」。時間を漫然と過ごすのではなく、一つ一つのセッションを自分の仕事に刻みつける。ポモドーロを回すたびに、成果が刻まれていく。

## 2. ポジショニング

### 市場の現状

| タイプ | 例 | 弱点 |
|--------|-----|------|
| シンプルタイマー | Focus Keeper, Be Focused | 無個性。差別化ゼロ |
| ゲーミフィケーション | Forest | 木を育てる楽しさはあるが、プロのツール感がない |
| 多機能プロダクティビティ | Session, Focus Flow | 機能過多。Pomodoro以上のことをやりすぎ |

### 空いているポジション

「道具としての美しさ」で勝負する、Things 3やBearのような"craft app"。Pomodoro appでここを本気で取りに行っているプレイヤーがいない。

### Kokukokuのポジション

**「何もない」ことが美しいアプリ。**

- ライカのように、必要な機能だけが完璧に動く
- 開いた瞬間に迷わない。タイマーがある。押す。集中する。終わる。それだけ
- 装飾ゼロ。アニメーションも最小限。余白とtypographyで品を出す
- 「日本的」を主張するのではなく、結果として日本的な引き算の美が滲む

## 3. 差別化の2本柱

### Native Craft

触った瞬間にわかる質。速さ、滑らかさ、OSとの一体感。

- SwiftUIでゼロから構築
- macOS/iOSの一部のように振る舞う

### Radical Simplicity

機能を足さないことが価値。

- タイマーを開始する。集中する。終わる。それだけ
- タスク管理はしない（Things / Todoistに任せる）
- チーム機能なし
- 複雑な統計・レポートなし
- 外部サービス連携なし
- AI機能なし
- 買い切りでサブスクなし——「余計なことをしない」の一貫した思想

## 4. スコープ段階（Phase）

### Phase 1（MVP）

- Focus / Short Break / Long Break の標準Pomodoro体験
- 25/5/15（デフォルト）+ 設定で調整可能
- 自動開始ON（デフォルト）+ ターム境界で停止できる制御
- 履歴画面（最低限の記録と閲覧）
- 通知：デフォルトは音あり、設定でサイレントに切替可能

### Phase 2（Native拡張）

- メニューバー常駐（Mac）
- ウィジェット / Live Activity
- Apple Watch対応
- Focus Mode連携（タイマー中は自動でDnD）
- **Generative Mode（ジェネラティブモード）**
  - タイマー画面をジェネラティブアートに切り替えるオプション（設定で ON/OFF）
  - 残り時間は小さく数字で常時表示。図形は時間を「感じる」ための表現
  - 入力パラメータは「残り時間」「セッション種別」「セッション番号」のみ。決定論的（同じ時刻は同じ描画）
  - 4セッション × 4種類の異なるビジュアルで、1サイクルが1つのストーリーになる:
    - **Session 1: 円の分割（Subdivision）** — 1つの円が時間経過で幾何学的に分割。三角形→六角形→多角形へ。0:00で結晶的な曼荼羅が完成。静かに立ち上がる
    - **Session 2: 線描画（Lissajous）** — 1本の線がリサージュ曲線を25分かけて描き切る。「刻む」のメタファー。軌跡が刻まれていく
    - **Session 3: グリッド変容（Grid Morphing）** — ドット/線のグリッドが回転・間隔・サイズ変化でモアレ効果を生む。最も没入感が深い中村勇吾的ゾーン
    - **Session 4: 同心円の呼吸（Breathing Rings）** — 同心円がゆっくり脈動。ロングブレイク前のクールダウン
  - **Break中の表現: 減衰（Decay）** — 完成した図形がBreak時間（5分/15分）をかけてゆっくり透明になり消えていく。次のFocus開始時にキャンバスがクリアになる。セッションの「生→消」がライフサイクルとして視覚化される
  - 技術: SwiftUI Canvas + TimelineView。Metal不要の軽量実装。バッテリー負荷を最小に

### Phase 3（仕上げ）

- UI/モーション/アクセシビリティの磨き込み
- onboardingと課金導線の最適化
- ストア向け素材とコピーの改善
